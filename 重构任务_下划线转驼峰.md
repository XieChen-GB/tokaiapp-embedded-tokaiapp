# 重构任务：Entity 字段名下划线转驼峰命名

## 背景

项目路径：`Y:\電算室\謝\Java\tokaiapp\tokaiapp_embedded\tokaiapp`

当前所有 Entity 类的成员变量都用下划线命名（如 `ji_cd`），不符合 Java 命名规范（应为驼峰式 `jiCd`）。需要重构为驼峰命名，但**数据库列名不能变**，靠 `@Column(name="...")` 注解维持原有映射。

- `entity/Tok1bukkId.java`

## 具体改动规则

1. **成员变量名**：下划线命名 → 驼峰命名
   - 例：`private String ji_cd;` → `private String jiCd;`

2. **`@Column(name = "...")` 注解的值绝对不能改**
   - 例：`@Column(name = "ji_cd") private String jiCd;` ← 注解值保持 `ji_cd` 不变，只改变量名

3. **getter / setter 方法名同步改为驼峰**
   - 例：`getJi_cd()` → `getJiCd()`，`setJi_cd()` → `setJiCd()`

4. **构造方法参数名同步改**（Id 类里有带参构造方法，参数名和内部赋值都要改）
   - 例：`Tok1ankkId(String jyu_ji_cd, String an_no)` → `Tok1ankkId(String jyuJiCd, String anNo)`

5. **`equals()` / `hashCode()` 方法里引用的字段名同步改**

## 连带影响：调用方同步更新

以下目录中，所有调用了上述 Entity / Id 类 getter、setter、构造方法的地方，必须同步更新为新的驼峰方法名：

- `controller/` 下所有文件
- `service/` 下所有文件
- `repository/` 下所有文件（如有自定义方法引用字段名）

请在改动前后，搜索这几个目录中对旧方法名（如 `getJi_cd`、`setAn_no` 等）的所有引用，确保无遗漏。

## 验证步骤

全部改完后执行：

```bash
./mvnw clean install -DskipTests
```

确认编译无错误。如有编译失败，根据报错信息定位遗漏的引用并修正，重新编译，直到通过为止。

## 注意事项

- 不要修改 `@Table(name = "...")`、`@Column(name = "...")` 等注解中的字符串值，这些是数据库实际的表名/列名。
- 不要修改 `@EmbeddedId`、`@Embeddable`、`@Entity` 等结构性注解。
- import 语句不需要改动。
